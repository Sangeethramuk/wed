Q1. Explain MVC architecture
MVC stands for Model-View-Controller. It is a software design pattern that separates an application into three componentsE1.
The Model handles data and business logic. It manages the database and responds to requests for information.
The View is responsible for displaying data to the user. It receives data from the Model through the Controller.
The Controller acts as an intermediary. When a user interacts with the View, the Controller processes the input and updates the Model accordingly.
In a banking application, the Model would handle account balances and transaction recordsE2. The View would display the account summary page. The Controller would handle the transfer request and update both the balance and the transaction history.
